Attention_Is_All_You_Need

RNN与LSTM

Transformer出现之前，RNN与LSTM是处理序列数据的主流模型

RNN/LSTM机制：想象一下我们是怎么阅读句子的：一个词一个词地按顺序读。RNN模型也是这样工作的，它会按顺序处理每个词，并维护一个“记忆”（隐藏状态），这个记忆会传递给下一个词的处理过程。 这种方式很符合直觉，但也带来一个巨大的问题：

它必须按顺序计算，无法并行处理。 就像你不能在读完第一个词之前就去读第五个词一样。这导致训练过程非常缓慢，尤其是当句子很长时。

Transformer

Transformer模型如何工作？

Transformer模型分为两部分，Encoder和Decoder

1.input部分：

  • Input Embedding: 计算机不认识文字，所以第一步是把每个词转换成一串数字（向量），这个过程叫“词嵌入”（Embedding）。

  • Positional Encoding (位置编码): 因为模型一次性看所有词，它本身是不知道词的顺序的。为了解决这个问题，研究者引入了“位置编码”。 它就像给每个词的“词嵌入”向量上，额外添加了一个能表示其在句子中位置（是第1个词、第2个词还是第n个词）的“位置标签”。 这样模型就能知道词语的顺序了。

2.Encoder部分（图左侧）：

编码器的作用是“理解”输入的句子。它由很多（论文中是6层 ）相同的层组成，每一层的工作如下：

  • Multi-Head Attention (多头注意力): 核心！模型在这里计算“自注意力（Self-Attention）”，也就是说，句子中的每个词都会“看”向句子中的所有其他词（包括自己），来计算哪些词对自己最重要，从而更好地理解自己的含义。

    为什么叫“多头”？ 与其只用一种方式去计算注意力，不如用多种方式（比如论文里用了8个“头” ）。这好比让8个不同的人分头去阅读和理解句子，每个人关注的重点可能不同（比如有人关注语法结构，有人关注语义关联），最后把8个人的理解综合起来，得到一个更全面、更丰富的表示。

  • Feed Forward Network (前馈神经网络): 在注意力层之后，信息会通过一个简单的前馈神经网络，进行进一步的加工和转换。

    在每个子层（注意力层和前馈网络）之后，都有一个“Add & Norm”的步骤，这是一种稳定训练过程的技术，能让模型更深、更稳定

3.Decoder部分（图右侧）：

解码器的作用是根据编码器对原文的理解，生成目标语言的句子。 它也由许多相同的层组成，但比编码器多了一个注意力层

  • Masked Multi-Head Attention (带掩码的多头注意力): 解码器在生成译文时，也需要使用自注意力。但这里有一个限制：在预测第i个词时，它只能“看到”前面已经翻译好的 1 到 i-1 个词，而不能“偷看”后面的答案。这个限制就是通过“掩码（Masking）”来实现的。

  • Multi-Head Attention: 这是解码器与编码器交互的地方。解码器中的每个词都会“关注”编码器输出的所有词，这使得它能够准确地将原文和译文对应起来。 例如，在翻译德语时，它能知道当前要生成的德语词对应原文中的哪个英语词。

  • Feed Forward Network: 与编码器中的作用相同。

4.output：

经过解码器栈的处理后，最后的向量会经过一个线性层和一个Softmax层，来预测下一个最有可能的单词是什么

自注意力机制为何如此强大？

论文p6的表格中对比了自注意力、RNN和卷积三种结构

  • 最大路径长度 (Maximum Path Length): 这是衡量两个词之间信息需要传递多少步。对于自注意力，任何两个词之间的路径长度都是1，因为它们可以直接相互“关注”。 而对于RNN，第一个词和最后一个词之间需要走 n 步。 更短的路径意味着模型能更容易地学习到长距离的依赖关系（比如句子开头的主语和结尾的谓语之间的关系）。

    在Transformer之前，像RNN或LSTM这样的模型处理文本的方式是“顺序的” 。这就像我们读书，一个字一个字地读，读到后面的时候，脑子里还留着前面的“记忆”。这种方式有两个主要缺点：

    • 计算无法并行，速度慢：因为必须处理完第t-1个词才能处理第t个词，所以无法将一个长句子的计算任务分配给多个处理器同时进行 。这在处理海量数据时成为了一个巨大的瓶颈。
    • 长距离依赖问题：对于很长的句子，模型在处理到句末时，可能已经“忘记”了句首的重要信息 。就像一个人读了一篇很长的文章，可能记不清开头第一段的具体细节了。
    • Transformer通过完全抛弃这种循环结构，并仅依赖自注意力机制，从根本上解决了这个问题。自注意力层可以一次性连接所有单词，任何两个单词之间的信息传递路径都只有1 。这不仅让大规模并行计算成为可能，极大地加快了训练速度 ，也让模型更容易捕捉到句子中相距很远的词之间的关系 。

  • 顺序操作 (Sequential Operations): 自注意力是 O(1)，意味着计算可以大规模并行。而RNN是 O(n)，必须一步一步来。 这就是Transformer训练快得多的根本原因。

多头注意力计算方法

先理解“单头”，再是“多头”。所谓单头，就是缩放点积注意力。它的核心是为句子中的每个词，计算出一个融合了全局上下文的新表示。为了实现这一点，每个输入的词向量都会被转换成三个不同的向量：

  • Query (查询向量, Q): 代表当前这个词，它要去“查询”和自己相关的其他词。
  • Key (键向量, K): 代表句子中可以被查询的词。可以把它看作每个词的“标签”或“索引”，用来和Query匹配。
  • Value (值向量, V): 代表词本身包含的真正信息。一旦Query和某个Key匹配上了，就从对应的Value中提取信息。

这三个向量都是由原始的词嵌入向量乘以不同的权重矩阵得到的，这些权重矩阵是模型在训练中学习到的。

单头计算过程（图左侧）：

  • 计算相关性得分 (MatMul): 将你的查询向量 (Q) 与所有词的键向量 (K) 进行点积运算 (QK^T) 。这个点积的结果就是一个分数，分数越高，代表Q和K的“相关性”或“相似度”越高。

  • 缩放 (Scale): 论文作者发现，当向量维度 d_k 很大时，点积的结果会变得非常大，这会导致后续Softmax函数的梯度变得极小，不利于模型训练 。因此，他们用一个“缩放因子”来稳定数值，即将上一步得到的所有分数除以 sqrt(d_k)，其中d_k 是键向量的维度 。这就是“缩放”的由来。

  • 归一化为权重 (Softmax): 将缩放后的分数输入到一个Softmax函数中 。Softmax会将这些分数转换成一组总和为1的百分比权重。相关性高的词会获得较高的权重（如0.99），不相关的词权重则趋近于0。

  • 加权求和 (MatMul): 将上一步得到的权重，分别乘以每个词对应的值向量 (V) 。最后将所有加权后的V向量相加，得到最终的输出。

这个输出就是一个新的向量，它很好地融合了句子中所有词的信息，并且是根据“注意力”权重来决定融合多少的。

多头计算过程：

理解了上面的“单头”注意力，多头就非常简单了。

论文作者认为，只从一个角度去计算注意力可能会限制模型的能力。就像一个人看问题可能会有盲点，多几个人从不同角度看会更全面。

多头注意力就是基于这个思想：

它不是进行一次注意力计算，而是把输入拆分成多份，并行地进行多次注意力计算，最后再把结果合并起来 。

  • 线性投射 (Linear): 假设我们要实现一个8头的注意力机制 (论文中 h=8 )。我们不直接使用原始的Q、K、V，而是准备8组不同的权重矩阵。原始的Q、K、V分别与这8组权重矩阵相乘，得到8套不同的Q、K、V 。这一步可以理解为，把原始问题“投射”到8个不同的“表示子空间”中，让每个头去关注不同的方面。

  • 并行计算注意力 (Scaled Dot-Product Attention): 8个头，各自独立地、并行地执行上面我们讲的“缩放点积注意力”计算，每个头都会得到一个自己的输出向量 。

  • 拼接 (Concat): 将8个头输出的8个结果向量拼接在一起，形成一个更长的向量 。

  • 再次投射 (Linear): 最后，将这个拼接后的长向量再乘以一个额外的权重矩阵 (W^O)，把它融合并降维回模型所需要的标准维度 。

好处在哪？

论文中提到，多头注意力允许模型在不同位置共同关注来自不同表示子空间的信息 。通俗地说，一个头可能学会了关注语法结构，另一个头可能学会了关注动词和宾语的语义关系，还有一个头可能关注了代词指代关系等等。将这些不同的关注点结合起来，模型就能对句子有一个远比单头注意力更丰富、更立体的理解。
